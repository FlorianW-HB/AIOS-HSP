# Architektur

## Grundprinzip

Das Repository ist so aufgebaut, dass gemeinsamer Kern und firmenspezifische Inhalte getrennt bleiben.

## Trennlinie

- `aios/shared`: Inhalte ohne firmenspezifischen Bezug
- `aios/hsp-recht`: Inhalte nur für HSP RECHT
- `aios/hsp-steuer`: Inhalte nur für HSP STEUER

## Ziel

Später sollen neue Agenten, Prompts und Arbeitsabläufe ohne Umbau ergänzt werden können.
